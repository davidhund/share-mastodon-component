<share-mastodon> — Voorbeelden

Een <share-mastodon> HTML Web Component. Opent een dialog en onthoudt je server.

Alle attributen op een rij:

  • server — stel de Mastodon-instantie hostname vooraf in
  • lang — forceer de taal van de UI (en / nl; standaard automatisch gedetecteerd)
  • show-icon — toon het Mastodon-icoon naast de linktekst
  • icon-only — toon alleen het icoon (tekst is visueel verborgen maar nog steeds toegankelijk); impliceert show-icon
  • text — aangepaste deeltekst (standaard document.title)
  • via — voeg een "via"-handle toe aan de deeltekst
  • label — overschrijf de linktekst
  • label-explainer — visueel verborgen toelichting wanneer er nog geen server is ingesteld
  • edit-text — label van de "server bewerken"-knop
  • dialog-label — koplabel van de dialog
  • dialog-hint — hinttekst in de dialog
  • dialog-hint-invalid — foutmelding bij ongeldige server; gebruik %SERVER% als plaatshouder
  • dialog-save — label van de opslaan-knop
  • dialog-cancel — label van de annuleren-knop

1. Minimaal — alleen tekstinhoud

De eenvoudigste vorm: wikkel de component om willekeurige tekst. De component maakt automatisch het hyperlink, de dialog en het stijlblad aan.

Deel via Mastodon 
<share-mastodon>Deel via Mastodon</share-mastodon>

2. Met een eigen <a>-hyperlink

Geef een bestaand hyperlinkelement mee als kind — de component omhult het en verbetert de href zodra een server bekend is. Handig als no-JS-fallback.

Deel via Mastodon 
<share-mastodon>
  <a href="https://share.joinmastodon.org/#text=Bekijk%20dit%20eens">Deel via Mastodon</a>
</share-mastodon>

3. lang — taal forceren

Overschrijf de automatische detectie om een specifieke taal af te dwingen. Ingebouwd: en, nl. Valt terug op nl voor onbekende talen (met een consolewaarschuwing).

Deel via Mastodon 
<share-mastodon lang="en">Deel via Mastodon</share-mastodon>

4. server — instantie vooraf instellen

Sla de dialog over door een standaard hostnaam op te geven. De link gaat direct naar de deelpagina. De waarde wordt ook opgeslagen in localStorage en gedeeld met alle instanties op de pagina.

Deel via Mastodon 
<share-mastodon server="mastodon.social">Deel via Mastodon</share-mastodon>

5. show-icon — met Mastodon-icoon

Voegt het Mastodon SVG-logo toe vóór de link. Het icoon heeft role="presentation" en is 1em groot.

Deel via Mastodon 
<share-mastodon show-icon="true">Deel via Mastodon</share-mastodon>

6. icon-only — icoon met visueel verborgen tekst

Toont alleen het icoon; de linktekst is visueel verborgen maar blijft zichtbaar in de accessibiliteitsboom. Impliceert show-icon.

Deel via Mastodon 
<share-mastodon icon-only="true">Deel via Mastodon</share-mastodon>

7. text + via — aangepaste deelinhoud

text overschrijft de standaard deeltekst (anders document.title). via voegt een handle toe. Beide worden opgenomen in de URL-gecodeerde Mastodon-deelstring.

Deel via Mastodon 
<share-mastodon
  text="Bekijk dit artikel!"
  via="@jij@mastodon.social"
  show-icon="true"
>Deel via Mastodon</share-mastodon>

8. Vertaalattributen — alle i18n-attributen

Elke tekst in de UI kan per instantie overschreven worden zonder de broncode aan te passen. Gebruik %SERVER% als plaatshouder in dialog-hint-invalid.

Deel deze pagina 
<share-mastodon
  label="Deel deze pagina"
  label-explainer="Voer eerst je Mastodon-serveradres in om te delen."
  edit-text="Mastodon-server aanpassen"
  dialog-label="Jouw Mastodon-server:"
  dialog-hint="bijv. mastodon.social, social.overheid.nl of mas.to"
  dialog-hint-invalid="'%SERVER%' lijkt geen geldige server."
  dialog-save="Ga verder"
  dialog-cancel="Annuleren"
>Deel deze pagina</share-mastodon>

9. Volledig voorbeeld — alle attributen

Alles gecombineerd: vooraf ingestelde server, icoon, aangepaste deeltekst en via-handle, én alle vertaaloverschrijvingen.

Deel via Mastodon 
<share-mastodon
  server="social.overheid.nl"
  lang="nl"
  show-icon="true"
  text="Bekijk deze Mastodon-deelwidget!"
  via="@jij@social.overheid.nl"

  label-explainer="Voer je Mastodon-server in om deze pagina te delen."
  edit-text="Server aanpassen"

  dialog-hint="bijv. mastodon.social, social.overheid.nl of mas.to"
  dialog-hint-invalid="'%SERVER%' lijkt geen geldige server."
  dialog-save="Ga verder"
  dialog-cancel="Annuleren"
>Deel via Mastodon</share-mastodon>

10. Custom events

De component vuurt drie custom events. Alle events bubbelen op en zijn cancelable. Ze worden altijd geprefixed met de tagnaam: share-mastodon:. Luister op document of op een bovenliggend element om alle instanties op de pagina af te vangen, of direct op het element voor één specifieke instantie.

share-mastodon:init

Wordt eenmalig afgevuurd zodra de component volledig geïnitialiseerd is. event.detail is het component-element zelf.

Deel via Mastodon 
document.addEventListener('share-mastodon:init', (event) => {
  console.log('Geïnitialiseerd:', event.detail); // het <share-mastodon> element
});

share-mastodon:dialog:open

Wordt afgevuurd op het moment dat de dialog wordt geopend. event.detail is een leeg object.

Deel via Mastodon 
document.addEventListener('share-mastodon:dialog:open', (event) => {
  console.log('Dialog geopend door:', event.target);
});

share-mastodon:dialog:close

Wordt afgevuurd nadat de dialog gesloten is. event.detail bevat de returnValue van de dialog:

  • hostnaam (bijv. "mastodon.social") — gebruiker heeft een geldige server opgeslagen
  • "cancel" — gebruiker heeft de dialog gesloten zonder op te slaan
  • "" (lege string) — dialog sloot op een andere manier (bijv. Escape of closedby="any")
Deel via Mastodon 
document.addEventListener('share-mastodon:dialog:close', (event) => {
  const returnValue = event.detail;

  if (!returnValue || returnValue === 'cancel') {
    console.log('Dialog gesloten zonder wijziging.');
    return;
  }

  console.log('Server opgeslagen:', returnValue); // bijv. "mastodon.social"
});
