<share-mastodon> — Interactive Examples

📋 Before you start

  1. Build the component: npm run build
  2. Open this file in a browser: npx serve . then navigate to http://localhost:3000/index.html
  3. Open your browser's Developer Console (F12 or Cmd+Opt+I) to see custom events being logged
  4. Interact with any example below to test the component's behavior

1. Minimal Usage

The simplest form: wrap text inside <share-mastodon>. The component automatically creates a link, dialog, and stylesheet.

Share via Mastodon 
<share-mastodon>Share via Mastodon</share-mastodon>

2. With Progressive Enhancement

Wrap an existing <a> tag for a no-JS fallback. The link works even without JavaScript.

Share via Mastodon 
<share-mastodon>
  <a href="https://share.joinmastodon.org/#text=Check%20this%20out">Share via Mastodon</a>
</share-mastodon>

3. Preset Server (Skip Dialog)

Use the server attribute to skip the dialog and go directly to the share page. The value is saved to localStorage and shared across all instances.

Share via Mastodon 
<share-mastodon server="mastodon.social">Share via Mastodon</share-mastodon>

4. With Icon

Use show-icon="true" to display the Mastodon SVG logo before the text. The icon is 1em in size and inherits the current text color.

Share via Mastodon 
<share-mastodon show-icon="true">Share via Mastodon</share-mastodon>

5. Icon Only (Accessible)

Use icon-only="true" to show only the icon. The link text is visually hidden but remains in the accessibility tree for screen readers.

Share via Mastodon 
<share-mastodon icon-only="true">Share via Mastodon</share-mastodon>

6. Custom Share Text & Via Handle

Use text to override the share message (defaults to document.title). Use via to append a handle mention.

Share this article 
<share-mastodon
  text="Check out this article!"
  via="@you@mastodon.social"
  show-icon="true"
>Share this article</share-mastodon>

7. All i18n Attributes

Every text in the UI can be customized per instance via attributes. Use %SERVER% as a placeholder in dialog-hint-invalid.

Share 
<share-mastodon
  label="Share this page"
  label-explainer="Enter your Mastodon server first."
  edit-text="Change server"
  dialog-label="Your Mastodon server:"
  dialog-hint="e.g. mastodon.social or fosstodon.org"
  dialog-hint-invalid="'%SERVER%' is not a valid server."
  dialog-save="Continue"
  dialog-cancel="Cancel"
  show-icon="true"
>Share</share-mastodon>

8. Locales (Multi-language)

The component ships with English built-in. Additional languages (Dutch, French, German, Spanish) can be loaded as optional locale files.

To enable other languages, load the locale file before creating component instances:

<script type="module" src="./dist/share-mastodon.min.js"></script>
<script type="module" src="./dist/locales/fr.js"></script>

Try switching languages:

Share 
<share-mastodon lang="en" show-icon="true">Share</share-mastodon>

11. Custom Events

The component emits three custom events. All events bubble and are cancelable. Listen on document to catch all instances, or on the element itself for a single instance.

Event #1: share-mastodon:init

Fired once when the component initializes. Check your browser console to see it logged.

Share via Mastodon 
document.addEventListener('share-mastodon:init', (event) => {
  console.log('Component initialized:', event.detail);
});

Event #2: share-mastodon:dialog:open

Fired when the dialog opens. Useful for tracking or analytics.

document.addEventListener('share-mastodon:dialog:open', (event) => {
  console.log('Dialog opened:', event.target);
});

Event #3: share-mastodon:dialog:close

Fired after the dialog closes. event.detail contains the result:

  • Hostname (e.g. "mastodon.social") — user saved a valid server
  • "cancel" — user clicked cancel
  • "" (empty string) — dialog closed via Escape or other means
Share and watch the console 
document.addEventListener('share-mastodon:dialog:close', (event) => {
  const returnValue = event.detail;

  if (!returnValue || returnValue === 'cancel') {
    console.log('Dialog closed without saving.');
    return;
  }

  console.log('Server saved:', returnValue);
});
